## Потоки
- Основой поток - main. Другие потоки - вторичные.
- Процесс - изолированное по виртуальной памяти приложение. Потоки могут передавать указатели.
- Клиенты это потоки.
## Приоритеты
- LOW_PRIORITY - клиент будет ждать, пока другие клиенты не завершат чтение таблицы
	- не используется в , так как теряется параллельность
- DELAYED - операция выполняется сразу же, запись добавляется сразу же, как таблицу перестанет использовать другой поток
	- используется в ISAM MyISAM
	- асинхронный режим передачи сообщений.
- IGNORE - все строки, дублирующие имеющиеся primary key или вторичные, будут проигнорированы.
	- Если его не указывать, т.е. ключи совпадают с уже имеющимися, операция прекращается на той вставке, где было найдено совпадение.

## Команда LOAD DATA
- Для импорта множества записей из текстового файла можно использовать команду LOAD DATA.
- LOAD DATA LOCAL INFILE "имя_файла " INTO TABLE имя_таблицы; 
	- Каждай строка отдельная запись
	- Строки разделены символами табуляции
	- Поля в том порядке, в каком они указаны в таблице
- Можно импортировать записи с частично пустыми полями(для случая когда мы не знаем, сколько полей). Используется \n
- ![[Pasted image 20251203154417.png]]
- Local указывает, что файл находится не на сервере, а на клиенте. Либо он доступен всем пользователям отовсюду.
	- Сервер не может прервать операцию при передаче данных.
	- 
- CONCURRENT - другие клиенты могут извлекать значения из таблицы при выполнении операции
	- MyISAM
- REPLACE - запись замещает существующую по ключу(если они совпадают). Аналогично для IGNORE.
	- Если ни одного нет, может возникнуть ошибка, все что дальле в файле игнорируется.
- ? "Операция выполняется в один так процессора"
- TEMINATED - указытель, чем разделяются строки и поля.
- BORDER - символ, обрамляющий поле с начала и конца.
- В качестве разделителя можно указывать несколько символов, а не один.

## Команда SELECT
- Вид расширенного select
![[Pasted image 20251203160912.png]]
- Ключевое слово HAVING. Как WHERE отличается порядком выполнения операций. 
- ![[Pasted image 20251203165103.png]]
- При этом случай использования ключевое слово where. Сначала производится выборка из таблицы с применением условия, лишь затем группировка резлуьтата.
- В случае использования HAVING сначала происходит группировка таблицы, а потом только выборка.
- В выражение WHERE можно использовать функции, поддерживаемы в MySQL. 
- HAVING может ссылаться на любой столбец или псевдоним и отрабатывается последним перед отправкой данных клиентом без оптимизаций.
