# Знакомство с MySQL. Создание базы данных и таблиц. Типы данных.
- База данных создается командой CREATE DATABASE <db_name>;
	- Максимальное количество символов в имени бд 64
	- нельзя использовать "/" и "."(слэш и точку)
	- ![[Pasted image 20251119151324.png]]
	- Данная команда создает только папку
- Бд mysql создается как каталоги с файлами, эти файлы соответствуют таблицам в бд.
- Чтобы использовать бд, подключиться к ней, нужна команда use <db_name>;
- Создание таблицы
	- Максимальное количество символов в имени бд 64
	- нельзя использовать "/" и "."(слэш и точку)
- ![[Pasted image 20251119151630.png]]
	- Присутствуют ограничения на переменные и возможность задать значения по умолчанию
	- ![[Pasted image 20251119151853.png]]
- В рамках создания бд лучше соблюдать верхний регистр при написании запросов для ключевых слов, в иных случаях следует использовать нижний регистр. 
	- Хотя имена бд и таблиц не чувствительны к регистру на windows, не следует ссылаться на конкретную бд или таблицу, используя различные регистры символов внутри одного и того же запроса. 
- Временная таблица, созданная с атрибутом TEMPORARY, автоматически удаляется при завершении соединения с базой. Ее имя(как идентификатор какого-то объекта) действительно только в рамках данного соединения(use), т.е. имя - локально для каждого отдельного соединения.

- В имени столбца можно использовать до 64 любых символов.
- Таблицы могут иметь разные типы:
- ![[Pasted image 20251119153349.png]]
	- InnoDB - поддерживает транзакции и блокировку на уровне строк, обеспечивая целостность данных по ACID. Это движок по умолчанию для большинства случаев. 
	- MyISAM - устаревший движок без поддержки транзакций, использует блокировку на уровне таблицы. Может быть быстрее InnoDB для нагрузок, ориентированных только на чтение
	- MERGE (не услышал)
	- MRG MyISAM - не движок хранения, а объединяющая таблица, которая представляет собой логическое представление нескольких одинаковых таблиц MyISAM.
	- HEAP - хранит все данные в оперативной памяти для максимально быстрого доступа. Все данные теряются при перезапуске сервера
	- BDB - поддерживает транзакции, но использует блокировку на уровне страниц, а не строк. Считается устаревшим и практически не используется. 
	- ISAM — прародитель MyISAM, использовался в самых ранних версиях MySQL. Полностью устарел и был заменен на MyISAM.
	- Все типы нужны для поддержания разных таблиц.
### Преимущества транзакционных таблиц
- Надежность. Если произойдет сбой в работе mysql, данные можно восстановить посредством автоматического восстановления или при помощи резервной копии и журнала транзакции.
- Можно сочетать(объединять) несколько операторов и обрабатывать их подтверждение командой COMMIT.
- Можно запустить команду ROLLBACK для отката внесенных и неподтвержденных изменений.
- Лучше обеспечивает параллельную работу при одновременных обновлении таблицы и чтении из нее.
### Преимущества нетранзакционных таблиц
- Работа с ними выполняется гораздо быстрее, т.к. не выполняются дополнительные транзакции.
- Требуется меньше дискового пространства.
- Для обновления таблицы используется меньшее количество памяти.
Сеть стабильно работает, всегда есть резервные копии, нет необходимости частого обновления - тогда использовать.
## Типы данных столбцов
![[Pasted image 20251119155223.png]]
- Вещественное число типа DECIMAL представляется собой вещественное число с фиксированной точностью, которое в отличие от типа DOUBLE хранится в строчном формате(каждая цифра - как строчный символ). M - общее количество  действительных цифр в разрядной сетке, D - количество действительных цифр после запятой.
- Для целых и вещественных можно использовать модификатор UNSIGNED для отбрасывания отрицательного диапазона в положительный.
- ![[Pasted image 20251119160627.png]]
- Тип ENUM('value1', 'value2') указывает на список возможных значений. Макс. число элементов 65535. 
	- Значение по умолчанию - первая величина из списка.
	- ![[Pasted image 20251119160718.png]]
- Тип SET('value1', 'value2'; ..). Макс. значение 64. Если значение типа enum должно содержать ровно 1 значение из указанного множества, то столбцы типа set могут содержать любой или все элементы заданного множества одновременно.
- Величина DEFAULT (по умолчанию), не может быть функцией или выражением - только константа. 
	- Если для данного столбца не задается значение по умолчанию, mysql автоматически назначает эту величину.
	- Если столбец может принимать NULL, то по умолчанию присваивается NULL.
	- Если столбец объявлен как NOT NULL, то значение по умолчанию зависит от типа столбца:
		- Для числовых типов(за исключением AUTO_INCREMENT) = 0. Для столбца с AUTO_INCREMENT значением по умолчанию является следующее значение в последовательности номеров, начинающихся с 1.
		- Для даты и времени из таблицы значения по умолчанию = соответствующей начальной величине для данного типа.
		- Для строковых типов, кроме enum, значение - пустая строка "".
- В некоторых случаях mysql без уведомления изменяет определение столбца, заданное командой create table:
	- Столбец varchar, с длиной меньше чем 4, преобразуется к типу char
	- Если некоторый столбец в таблице имеет переменную enum, то вся строка в результате будет иметь переменную enum. Если таблица содержит любые столбцы переменной длины(varchar, text), то все столбцы типа char, с длиной >3, автоматически преобразуются в столбцы типа varchar.
	- Mysql приводит в соответствие определенные типы столбцов, используемые другими производителями баз данных.
- Для просмотра структуры таблицы(из чего она состоит) с отображением столбцов и их типов можно использовать команду DESCRIBE <db_name>
## Основные типы ограничений
![[Pasted image 20251119165828.png]]
- Ограничения - это правила, которые применяются к данным в таблице для обеспечения их целостности и корректности.
	- Наличие нескольких одинаковых пользователей
	- Ссылки на несуществующие записи
	- Отсутствие обязательных данных
	- Ввод некорректных значений
- Первичный ключ должен быть NOT NULL и UNIQUE.
	- ![[Pasted image 20251119170050.png]]
	- Если первичный ключ состоит из одного атрибута, он - простой. Если состоит из 2 и более атрибутов, называется составным.
		- ![[Pasted image 20251119170302.png]]
- Внешний ключ - столбец или набор столбцов таблицы, который ссылается на первичный ключ другой таблицы. Нужен для обеспечения ссылочной целостности, при которой значения в столбце внешнего ключа соответствует значениям из столбца первичного ключа связанной таблицы.
	- ![[Pasted image 20251119170432.png]]
	- В данном примере ссылочная целостность обеспечивается следующими факторами:
		- Нельзя добавить чек для несуществующего пользователя.
		- Нельзя добавить пользователя, у которого есть товарные чеки.
	- Будем называться родительской таблицей ту таблицу, на первичный ключ которой ссылается внешний ключ другой таблицы, которую, в свою очередь, будем называть дочерней.
