# Процедуры
- Сохраненная процедура - объединение последовательности запросов с сохранением ее под определенным именем на сервере.
	-  Повторное использование кода и автоматизация рутины.
	- Сокращение сетевого трафика.
	- Безопасность - только привилегированный пользователь может выполнять процедуры
	- Простота доступа за счет инкапсуляции сложного кода внутри процедуры.
	- Выполнение бизнес логики. Можно перенести код из прикладной программы на сервер бд. Называется такой код переносимым. Используется с термином миграция.
	- Аналог - бат файлы.
- Класс - множество объектов с похожими свойствами и методами
- Создание процедуры
![[Pasted image 20251217152132.png]]
- Параметры не обязательны. 
	-  **`IN`** — это параметр для **передачи данных внутрь процедуры**, и его значение нельзя изменить внутри процедуры.
	-  **`OUT`** — это параметр для **возврата данных из процедуры** наружу, и его начальное значение внутри процедуры равно `NULL`.
	-  **`INOUT`** — это **комбинированный параметр**, который позволяет передать значение в процедуру, а затем вернуть из неё изменённый результат.
- При работе с хранимыми процедурами символ ";" в конце запроса воспринимается консольным клиентом как сигнал к отправке запроса на сервер. 
	- Переопределить разделитель запросов чтобы выполнять всю процедуру за раз на сервере.
![[Pasted image 20251217152931.png]]
- В рамках процедуры можно использовать ветвление и циклы.
# Ветвление и циклы
- Данные конструкции лучше обрамлять в begin end.
- Всегда должны быть условия выхода из цикла.
## Условные операторы
1. **IF-THEN-ELSEIF-ELSE** — классическое ветвление
2. **CASE** — множественный выбор по значению выражения
![[Pasted image 20251217153250.png]]
## Циклы
1. **WHILE** — цикл с предусловием (проверка в начале)
2. **REPEAT** — цикл с постусловием (выполняется хотя бы раз, проверка в конце)
3. **LOOP** — бесконечный цикл (требует явного выхода через `LEAVE`). Участвует в режиме прослушивания.
	1. Режим прослушивания - это состояние сервера, когда он ожидает входящих подключений от клиентов на определенных сетевых портах. Сервер "слушает" эти порты и принимает соединения, когда клиенты пытаются подключиться.
		1. **Порт** — виртуальный "канал" для сетевого соединения (0-65535)    
		- **Сокет (socket)** — конечная точка соединения (IP-адрес + порт)
		- **Bind (привязка)** — процесс связывания сервера с конкретным IP-адресом и портом
## Управление выполнением
- **LEAVE метка** — полный выход из цикла/блока, помеченного меткой (аналог `break`).
- **ITERATE метка** — переход к следующей итерации цикла (аналог `continue`).
- **GOTO метка** — безусловный переход (использовать с осторожностью)
	- Упрощают сложную логику переключения

# Триггеры
- Триггеры — это специальные хранимые процедуры, автоматически выполняющиеся при определенных событиях в таблице. Такая же хранимая процедура, но привязанная к событию изменения содержимого конкретной таблицы(EAC).
	- Не допускается привязывать к временной таблице/представлению.
	- Для таблицы может быть создан только один триггер для каждого из событий trigger_event и trigger_time. Всего 6 триггеров для таблицы.
- В MySQL нельзя привязать к сквозному(каскадному) обновлению или удалению записи таблицы типа InnoDB со связью первичный и вторичный внешний ключ.
![[Pasted image 20251217153821.png]]
- **Время срабатывания (trigger_time):**
	- `BEFORE` — перед выполнением операции
	- `AFTER` — после выполнения операции
- **События (trigger_event):**
	- `UPDATE` — при обновлении
	- `DELETE` — при удалении
	- `INSERT` — при вставке
- **Доступ к данным:**
	- `NEW.поле` — значение поля в новой записи (для INSERT/UPDATE)
	- `OLD.поле` — значение поля в старой записи (для UPDATE/DELETE)
![[Pasted image 20251217154533.png]]

- **Пример использования:** Автоматическое логирование изменений, проверка/коррекция данных перед записью, поддержание целостности.

# Транзакции
- Транзакция — атомарная последовательность SQL-операций, которая либо выполняется полностью, либо не выполняется совсем(как единая, непрерываемая операция). Не прерывается другими клиентами.
	- ISAM MYISAM HEAP(ахип) не поддерживают транзакции.
- Если часть транзакции выполняется со сбоем, результаты выполнения всех операторов транзакции до места сбоя - отменяются, бд приводится к виду до выполнения транзакции. Что за принцип ACID?
- По умолчанию mysql работает в режим автоматического завершения транзакций (автосохранения, autocommit). Сразу после выполнения обновления данных изменения тут же сохраняются (на диске).
	- Для объединения операторов в транзакцию следует отключить аутокоммит.

# Представления
- Представление — виртуальная таблица, результат SELECT-запроса. Дополнительный способ получения данных, в рамках которого запрос можно сохранить под уникальным именем и в дальнейшем вызывать как хранимую процедуру.
- Клиенту можно предоставить права доступа только к представлениям, но не к исходным таблицам.
- View отображается в списке таблиц бд. (show tables;)
- Представления без дополнительных столбцов - **вертикальные**
- Представления, отображающие только те строки, с которыми работают определенные пользователи - **горизонтальные**

![[Pasted image 20251217160218.png]]
- **Алгоритмы:**
	- `MERGE` — объединение запроса представления с внешним запросом
	- `TEMPTABLE` — создание временной таблицы
	- `UNDEFINED` (по умолчанию) — СУБД выбирает оптимальный алгоритм
- **Назначение:**
	- Упрощение сложных запросов
	- Ограничение доступа к данным
	- Обеспечение обратной совместимости

# Администрирование
![[Pasted image 20251217161200.png]]
- Администрирование - создание пользователей с разными правами доступа.
- root создается автоматически
	- `localhost` — доступ только с локальной машины
	- `%` — доступ с любого хоста (менее безопасно)
- Пароль при создании можно не указывать. По умолчанию будет пустая строка
	- Пароль вероятнее нужно хранить в хэш коде.

### Система привилегий:
**Основные привилегии:**
- `SELECT`, `INSERT`, `UPDATE`, `DELETE` — операции с данными
- `CREATE`, `DROP`, `ALTER` — управление структурой
- `GRANT OPTION` — право предоставлять привилегии другим
- `ALL [PRIVILEGES]` — все привилегии (кроме `GRANT OPTION`)
### Все возможные привилегии
![[Pasted image 20251217161715.png]]
**Уровни привилегий:**
1. **Глобальный** (`ON *.*`) — все базы данных на сервере
2. **Уровень БД** (`ON база.*`) — все объекты конкретной базы
3. **Уровень таблицы** (`ON база.таблица`) — конкретная таблица
4. **Уровень столбцов** (`ON база.таблица(столбец)`) — отдельные столбцы
![[Pasted image 20251217161824.png]]

**Принцип минимальных привилегий:** Предоставлять пользователям только те права, которые действительно необходимы для их задач.
