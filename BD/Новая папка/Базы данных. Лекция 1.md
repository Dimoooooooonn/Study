Вопросы: что такое логическое и физическое представление; что такое машинный код; типы баз данных;
## Введение из пролога
- На ряду с интерпретаторами и компиляторами используются трансляторы совмещенного типа, интерпретаторы-компиляторы:
	- На этапе разработки и отладки транслятор работает как интерпретатор.
	- После отладки исходная программа транслируется в объектный модуль(в режиме компиляции).
# Введение в базы данных
- Схема бд - представление моделей бд. Бывает логической(и) или физической(и).
- Любая база данных - хранилище данных в виде записей.
	- Запись - контейнер, в котором группируются свойства объекта/объектов(характеристики). Запись - содержательная структура описывающая элемент.
	- Записи подобны структурам из C/C++
	- Описываются поля с типом и размером.
	- Класс - множество объектов, имеющих аналогичные атрибуты и методы; структуры только атрибуты.
## Базовые модели баз данных
1. Иерархическая(древовидная) 
2. Сетевая
3. Реляционная
4. Объектно-реляционная
5. Объектно-ориентированная
6. Не реляционная
### Иерархическая модель
- Данные представлены в виде иерархической(древовидной) структуры.
- Суть модели в формировании связей между записями в виде дерева, с вводом соотношений типа родительский-дочерний(потомок) элемент.
	- Указатель - переменная, содержащая адрес ссылочного значения.
![[Pasted image 20250924154812.png]]
- Каждый элемент связан только с одним, вышестоящим(родительским) элементом. При этом на него могут ссылаться 1 или несколько элементов более низкого уровня.
- Схема не всегда удобна и часто избыточна.
	- Если в различных узлах автомобиля используются 300 элементов 1го типа, вместо 1 записи будет 300 дублирующих друг друга записей.
	- Для устранения избыточного дублирования, введена сетевая модель представления данных.
### Сетевая модель
- В сетевых бд используются представление о данных в виде произвольного графа.
![[Pasted image 20250924154300.png]]
- Состоят из вершин и ребер.
- Графы ориентированные(есть направление связей) и неориентированные.
- **Базовые свойства**:
	- Нет понятия уровней.
	- Объект может выступать как главный и подчиненный: имеет любое количество взаимосвязей. 1 запись может участвовать в нескольких отношениях родитель-потомок.
	- Допустимы связи между объектами на одном уровне.
- Поиск сложен, из-за наличия большого числа связей для определения маршрута.
### Реляционная
- Все данные сгруппированы в таблице.
	- Строки - это записи.
	- Столбцы - поля записей.
- Для однозначного перехода к конкретной записи, в каждой строке есть ключ, называем первичный ключ - уникальный идентификатор(идентификационный номер) записи.
	- Первичный ключ вводиться явно в отдельном столбце или неявно - автоматически самой СУБД.
	- В Oracle у любой строки есть скрытый уникальный столбец ROW_ID.
![[Pasted image 20250924160537.png]]
- Отношения родитель-потомок реализуются с помощью внешних ключей.
	- Внешний ключ - столбец, значения которого совпадают со значениями(например) первичного ключа другой таблицы.
- В таблицах в качестве признака отсутствующего значения используется значение NULL.
- Из-за возможного размещения константы NULL внутри полей записи, ни одно значение NULL != другому значению NULL по внешнему признаку.
![[Pasted image 20251008150652.png]]
- Основные понятия:
	- Атрибут - наименьший именованный элемент данных, характеризуемый именем и значением. Поле записи и его значение
	- Домен - допустимое множество значений заданного типа
	- Схема отношения - набор заголовков полей записей
	- Структурная схема бд - набор схем отношений
	- Арность - количество заголовков(столбцов) в схеме отношений
	- Кортеж - запись в таблице, определенная с точностью до соотношения имя атрибута и значение, и содержит 1 вхождение каждого имени атрибута, принадлежащего конкретной схеме отношения.
		- Значение должно соответствовать типу данных или домену
	- Отношение - множество кортежей, соответствующих одной схеме отношения.
	- (не использовать слово таблица, вместо него множества, количество)
