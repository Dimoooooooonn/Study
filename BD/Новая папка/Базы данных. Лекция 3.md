# Графовое хранилище
Графовое хранилище - сетевая бд, использующая узлы и ребра для отображения и хранения данных.

Узел содержит 1 значение и 1 указатель.
Указатель, переменная содержащая адрес памяти содержащий значение. Адреса хранятся в шестнадцатиричной системе счисления(как кодировать, декодировать).
В двумерном списке элемент содержит ссылку на след и пред.

Графовые бд подходят для обозначения двусложных связей между узлами и упрощают поиск взаимосвязей и закономерностей.

# СУБД
субд - посредник, с помощью которого пользователь взаимодействует с данными
## Функции:
1. предоставляет языковые средства описания данных и управления данными в основной(оперативка) и вторичной памяти(внешние накопители)
2. обеспечивает поддержку логических моделей данных.
3. поддерживает возможность создания логических данных и манипулирования ими(выбор, вставка, удаление, (crud)), а также одновременное отображение(выполнение) этих операций над физ. данными.
4. защита, обеспечение целостности(согласованности) данных, с поддержкой журналирования изменения, резервного копирования и восстановления после сбоев

## Работа с таблицей
1. Создание, модификация, удаление. Создание и модификация схем взаимосвязи существующих таблиц.(с примерами)
2. Ввод данных в таблицу - непосредственно или через форму(ui) и валидация вводимых значений
3. Поиск данных по критериям(выполнение поисковых запросов).
4. Создание отчетов по содержимому баз данных

# ACID
субд должна обладать свойствами ACID, чтобы работа с данными была надежной, предсказуемой и удобной:
1. Атомарность - операции проходят полностью или не проходят вообще. Предотвращает присутствие неполноценных записей в бд.
Пример: при оформлении заказа клиент не соглашается с офертой или не проходит оплата - весь процесс отменяется: данные не попадают в бд в неполноценном виде.
2. Согласованность(consistency) - после выполнения любой операции бд остается в полноценном состоянии с сохранением всех связей, ограничений и правил.
3. Изолированность - даже если несколько пользователей обращаются к бд одновременно, каждая операция выполняется в монопольном режиме 
4. Долговечность(надежность) - после записи данные остаются в бд. (пример: выключили питание, упал сервер)

## Дополнительные качества субд:
1. Масштабируемость - возможность обрабатывать с течением времени все большего объема данных без потери производительности
2. Доступность - беспроблемный доступ к бд
3. Безопасность - ограничение доступа( в том числе политик безопасности), шифрование, защита от взлома и утечек.

## 3 составные части субд:
1. Ядро - набор управляющих модулей(программ), предназначенных для автоматизации всех процессов, связанных с обращением к бд.
Постоянно находится в ОЗУ при работе с б'ами д'ых, т.е. является ее резидентом и организует обработку поступающих пользовательских запросов, управляет их очередностью выполнения, взаимодействует с прикладными программами и ОС, выдает комментарии и сообщения. 
Отвечает за управление данными во внешней и оперативной памяти и логгирование.
2. Набор обрабатывающих программ(handler)
3. Сервисные программы
### Набор обрабатывающих программ
Включает в себя трансляторы с  языков описания данных, схемы хранения, трансляторы с автономных языков программирования и интерпретатор с языка запросов sql.
**Сервисные программы(внешние утилиты)** предоставляют ряд дополнительных возможностей по обслуживанию информационных систем.

# Классификация субд по параметрам:
1. Хранение баз данных.
Локальные - все элементы субд(код) и баз данных находятся на одном сервере.
Распределенные - элементы субд находятся на разных серверах в т. ч. облачных.
2. Хранение и обработка данных и запросов.
- Клиент-серверные субд - субд и базы данных на одном сервере, к которому обращаются с запросами разные пользователи(клиенты). Получить данные можно с любого компьютера и специализированное ПО не требуется.
- Файл-серверные - базы данных хранятся на одном файл-сервере, т.е. выделенном компьютере для хранения файлов(данных) и поддерживающего совместный доступ к этим ним. Субд располагается на каждом устройстве, с которого отправляются запросы к базе данных. Чтобы пользователь мог получить доступ к данным на его должна быть установлена и настроена субд. Такие схемы используют для локально-корпоративных сервисов с хранением данных о клиентах компании и документообороте.
