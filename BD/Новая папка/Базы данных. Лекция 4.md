## Типы субд по архитекутуре
![[Pasted image 20251105151415.png]]
- Локальная(встраиваемая) субд - субд, которая встраивается в приложение как отдельный модуль и используется для управления данными внутри этого модуля.
- NoSQL - нереляционные субд, поддерживающие языки запросов, основанные на языках программирования, отличных от sql.
	- Применяется для работы с большими объемами данных, с архивами данных, с документами с большим количеством связей.
	- Oracle NoSQL используется схема ключ-значение.
- NewSQL - отказ от ресурсоемких функций SQL в пользу повышения производительности. Используются распределенные транзакции, позволяющие данным автоматические распределяться между серверами без потерь. Параллельные методы выполняются на разных узлах. Многие решения применяют колоночные схемы хранения данных.
	- Распределенная архитектура и расширенные функции newsql субд могут усложнить настройку, обслуживание, устранение неполадок(по сравнению с традиционными реляционными субд).
	- Золотая середина между SQL и NoSQL.
	- Ресурсоемкость - характеризуется величинами потребления оперативы и временем выполнения приложения.
	- Сложность - порядок зависимости времени и памяти необходимый для выполнения приложения от размерности входных данных.
	- Яйца? имеют толстые стенки???
	- Функционирование различных узлов как единого целого - кластеризация. Кластера не всегда выполняет параллельные вычисления.
- NXD (native xml database) - субд работает с xml как с основной структурой данных.
	- Используется в мед учреждениях
- Векторная субд - контейнеры, с данными из которых работа выполняется не как с обычным текстом или числом, а в векторном формате embeding'а: формате набора чисел, представляющего данные в числовой форме. При этом данные  и запрос пользователя в векторную форму переводит нейросеть.
	- Используется в машинном обучении(нейросети), рекомендательные системы
	- Нейросеть имитирует прием и обработку данных человеческого мышления. Мозг состоит из нейронов - клеток, способных распространять электрохимические сигналы. Нейрон имеет разветвленную структуру "входных каналов информации" - дентритов, ядро, аксон. Аксоны связывают нейроны с помощью синапсов.
		- Нейрон активируется когда суммарный уровень сигналов, поступивших в его ядро из дендритов, превышает определенный уровень, называемый порогом активации. Под входным источником можно воспринимать источник напряжения. Сумматор - рецептор.
		- Нейронная сеть может быть с учителем, так и без.
			- Учитель - мы точно знаем предварительные результаты(эталоны), которые должны получить от нейронной сети.
			- Оценка выходных результатов по критериям до тех пор, пока не получим ответ удовлетворяющий погрешности - без учителя
	- Суть - быстрый поиск по смыслу(или семантический поиск). Поиск по схожим векторам, а не по точным совпадениям. Семантика - смысловая нагрузка. 
	- Хорошо подходят для неструктурированных баз данных, обеспечиваю высокую масштабируемость.
- Бд временных рядов - хранят информацию, записываемую в хронологическом порядке.
	- Для графиков транспорта, пробок, производства.
	- Хорошо справляется с непрерывным потоком информации.
- Поисковые - созданы для быстрого поиска в больших объемах неструктурированных данных с поддержкой индексирования, когда каждое слово/фраза получает отдельный индекс.
	- Используется в антиплагиате
- Оперативные - хранят всю информацию в оперативке, без обращения к вторичной памяти.
## Наиболее распространенные (по использованию) субд
- Одной из самых распространенных является MySQL
	- Открытая(open source) реляционная субд. 
	- Бесплатная, простая настройка, скорость работы и гибкость - поддержка большого количества типов таблиц. Интерфейс простой, есть возможность синхронизации с другими бд, при этом функциональные возможности уступают PostgreSQL по работе с большими и сложными бд.
- Microsoft SQL Server
	- Реляционная субд. Бесплатная версия подходит для данных до 10гб. Используется в крупных компаниях по лицензии.
	- Есть импорта данных из других программ microsoft.
	- Высокая стоимость лицензии, повышенная ресурсоемкость.
- PostgreSQL. 
	- Бесплатная объектно-реляционная субд со встроенной функцией наследования таблиц, которые представлены в субд как объекты, действия с которыми выполняются с помощью ООП функций.
		- Это позволяет более сложную структуру данных.
	- Позволяет обрабатывать файлы такого типа как JSON XML. 
	- Субд поддерживает расширение sql на таких языках как python, java и т.д.
	- Преимущество: нет ограничений по размеру бд и числу записей в таблицах.
		- Используется в больших компаниях, где объем данных несколько петабайт
- Oracle - объектно-реляционная
	- Минусы: высокая стоимость лицензии. 
	- Используют компании, работающие с огромными объемами данных.
- SQLite
	- Компактная субд для реализации небольших проектов(сайтов, приложений) с ограниченным трафиком и объемом данных.
	- Работает везде: пк, телефон, тв, беспилотники. Не требуя администрирования и поддержки.
	- Встраивается в структуру библиотеки приложения с сохранением данных на одном устройстве
	- В качестве языка запросов использует язык Си
- MongoDB
	- Документно-ориентированная субд, данные хранятся как JSON файлы.
	- В качестве языка запросов используется собственный язык, базирующийся на js
	- Данные хранятся в виде коллекций документов вместо таблиц.
	- субд подходит для работы с большими объемами данных разных типов, причем данные автоматически распределяются между разными базами так, чтобы оптимизировать нагрузку и ускорить обработку запросов.
