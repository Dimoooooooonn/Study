- регистрация
- вход и выход
- восстановление пароля
- создание бронирования
- личный кабинет
- редактирование профиля
- обращения в поддержку
- отзывы
- простая обработка оплат
- сохранение данных в локальной базе проекта


После создания бронирования сервер сразу создаёт платежи:

- предоплата
- остаток

На странице `cabinet/payments.html` можно нажать кнопку **«Оплатить»**.
Это учебная имитация оплаты: статус меняется без подключения настоящего банка.

После оплаты:

- обновляется статус самого платежа
- обновляется статус бронирования
- меняются суммы «внесено» и «к доплате»
- в обзоре кабинета меняется статус оплаты

## Где теперь хранятся данные

Данные сохраняются в файле:

`data/sanatorium.db`

В ней сохраняются:

- пользователи
- бронирования
- платежи
- отзывы
- обращения в поддержку

## Запуск

```bat
start.bat
```
## Ручной запуск

```bash
mkdir out
javac -encoding UTF-8 -d out src/sanatorium/*.java
java -cp out sanatorium.Main
```

## Если порт занят

```bash
java -cp out sanatorium.Main 9090
```

И потом открыть:

`http://localhost:9090`

## Какие файлы самые важные

- `src/sanatorium/Main.java` — запуск сервера
- `src/sanatorium/SanatoriumServer.java` — маршруты и основная логика
- `src/sanatorium/DatabaseRepository.java` — работа с локальной базой проекта
- `src/sanatorium/DatabaseData.java` — структура данных базы
- `src/sanatorium/Payment.java` — модель платежа
